This repo contains the infrastructure necessary to deploy HTTPS TDX applications.

This project has several components:

docker_compose.yml - A Docker Compose file for VM deployments, using Traefik as the reverse proxy.

attestd - An attestation service that generates the TLS key inside the workload, obtains a Let's Encrypt certificate with DNS-01, and attests the served certificate.

echo - A simple example application docker image - echo service

client/browser - An HTTPS client for browser clients that virtualizes HTTPS to confirm the main client is using an HTTPS connection to the right certificate.

client/native - A client for native clients, that manually downloads the ssl certificate, attests it, and then pins it for connections. 

client/node - A Node.js client verifier for attested TLS certificates. It verifies
raw TDX quotes, dstack/Phala attestation metadata, and GCP Confidential Space
tokens, then prints the certificate public-key pin to use for future HTTPS
requests.

client/tdx_quote_verifier - A small local Rust verifier used by the Node client
for raw TDX quote verification.


# How to use

1) Set the deployment domain and ACME email.

```sh
export SERVER_DOMAIN="example.com"
export ACME_EMAIL="ops@example.com"
```

2) Start the VM or Compose deployment. Before issuance, Traefik serves its default certificate; this is only for bootstrapping the ACME DNS-01 flow.

3) Ask attestd to create an ACME order. The TLS private key is generated by attestd inside the workload.

```sh
curl -kfsS "https://$SERVER_DOMAIN/attestd/acme/start" \
  -H 'content-type: application/json' \
  --data "{\"domain\":\"$SERVER_DOMAIN\",\"email\":\"$ACME_EMAIL\"}" | jq .
```

If `ACME_ADMIN_TOKEN` is set, add:

```sh
-H "authorization: Bearer $ACME_ADMIN_TOKEN"
```

4) Create the returned TXT record in DNS:

```text
_acme-challenge.example.com. TXT <dns_value>
```

5) After DNS has propagated, finalize the order.

```sh
curl -kfsS "https://$SERVER_DOMAIN/attestd/acme/finalize" \
  -H 'content-type: application/json' \
  --data '{"order_id":"<order_id>"}' | jq .
```

The finalized certificate is written to the managed certificate path, and Traefik reloads it from its dynamic cert config.

6) Verify and pin the attested TLS certificate.

```sh
node external/tdx_easy_https/client/node/verify_client_tdx.mjs \
  "https://$SERVER_DOMAIN" \
  --gcp-confidential-space \
  --tls-domain "$SERVER_DOMAIN"
```

The first verification establishes that the certificate public key came from the attested workload. After that, clients can pin the printed `attested_tls_pin` until the server restarts or migrates and a new certificate key is generated.
